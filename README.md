# LP-Eval

LP-Eval is the public companion repository for the ICAIL 2026 accepted paper, *LP-Eval: Rubric and Dataset for Measuring the Quality of Legal Proposition Generation*. This repository is intended to release the code, data, and supporting materials associated with the paper.

LP-Eval is designed to support research on evaluating legal proposition generation. The repository centers on the evaluation rubric introduced in the paper and the accompanying dataset and implementation used to study generation quality in legal settings.

## Paper

This repository accompanies the following paper:

**LP-Eval: Rubric and Dataset for Measuring the Quality of Legal Proposition Generation**  
Accepted to **ICAIL 2026**

If you use this repository, please cite the paper using the citation information below.

## Repository Contents

The repository is organized around the materials needed to support the paper's release. As the public artifact is finalized, it is expected to include:

- evaluation code used for the paper's experiments and analysis,
- released data and supporting materials associated with LP-Eval,
- documentation needed to understand and use the released resources.

The repository currently remains intentionally lightweight, and this README is written to reflect the release structure without inventing files, scripts, or interfaces that are not yet present in the repository.

## Data

The `data/` directory is intended to contain the released dataset and any related materials described in the paper. Depending on the final release packaging, this may include data files, annotation artifacts, or documentation needed to interpret the dataset.

Users of the data should refer to the dataset materials provided in this repository for the authoritative release associated with the paper.

## Code and Evaluation

This repository is intended to contain the implementation used to support the experiments and evaluation presented in the paper. That includes the code required to work with LP-Eval and reproduce the evaluation workflow described in the publication.

Because the repository is still being finalized, this README does not claim specific commands, dependencies, or entrypoints that are not yet available in the current repo state. Those details should be documented alongside the released code once they are present.
